---
title: "Quickstart"
description: "Log in with the LogBrew CLI, set up SDK ingestion, and read logs, issues, actions, traces, and releases as JSON."
---

This quickstart gives you the shortest useful path: log in, prepare SDK setup,
send telemetry with a release, and read it back from the CLI.

Telemetry is the structured data your app sends to LogBrew, such as logs, trace
spans, issues, actions, releases, and environments.

<Note>
  Use `--json` whenever an AI coding agent needs stable output. Human-readable
  output is for terminal browsing.
</Note>

<Steps>
  <Step title="Install the CLI">
    Install the native `logbrew` CLI for your platform. If you build from
    source, install the public CLI package with Cargo.

    ```bash
    cargo install --git https://github.com/LogBrewCo/cli logbrew-cli
    ```

    Then verify that the `logbrew` binary is on your path and can print stable
    JSON for agents.

    ```bash
    logbrew version --json
    logbrew --help
    ```

    The JSON output should include the installed version, operating system, and
    architecture. The help output should list the main read targets: `logs`,
    `issues`, `actions`, `releases`, `trace`, and `issue`.
  </Step>

  <Step title="Log in once">
    Start browser-based login from the terminal.

    ```bash
    logbrew login
    ```

    After this step, humans and coding agents can use the CLI without opening a
    dashboard for every investigation.
  </Step>

  <Step title="Set up telemetry for your project">
    Ask LogBrew to plan SDK setup for the current project.

    ```bash
    logbrew setup --auto --yes --json
    ```

    Keep the release name stable and explicit, such as `api@1.2.3` or a git SHA.
    LogBrew rejects telemetry without a release and environment because
    release-level debugging depends on both fields.
  </Step>

  <Step title="Send a native telemetry envelope">
    First-party SDKs send one envelope that can contain logs, issues, trace
    spans, and actions. Every envelope belongs to one project, service, release,
    and environment.

    ```json telemetry-envelope.json
    {
      "project_id": "00000000-0000-0000-0000-000000000000",
      "environment": "production",
      "release": "api@1.2.3",
      "service_name": "checkout-api",
      "trace_id": "4bf92f3577b34da6a3ce929d0e0e4736",
      "sdk_name": "logbrew-rust",
      "sdk_version": "0.1.0",
      "items": [
        {
          "type": "log",
          "data": {
            "severity": "error",
            "message": "payment provider timeout",
            "source": "runtime",
            "attributes": { "route": "/checkout" },
            "occurred_at": "2026-06-01T12:00:00Z"
          }
        }
      ]
    }
    ```
  </Step>

  <Step title="Read telemetry back">
    Start broad, then filter.

    ```bash
    logbrew read releases --environment production --json
    logbrew read logs --release api@1.2.3 --environment production --json
    logbrew read issues --status unresolved --json
    logbrew read actions --name checkout_failed --since 24h --json
    logbrew read trace 4bf92f3577b34da6a3ce929d0e0e4736 --json
    ```
  </Step>
</Steps>
